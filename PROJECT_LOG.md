## 🏗️ 核心架构关键点

### 1. 游戏配置集中管理 (`CONFIG` 对象)
**位置**: `game.js` 顶部
```javascript
CONFIG = {
    canvasWidth: 480, canvasHeight: 640,
    plateSpeed: 8,
    giftTypes: [/* 三种礼盒配置 */],
    upgrades: { doublePlate: 500, triplePlate: 10000 }
}
```
**为什么重要**: 所有平衡参数集中在这里，调整游戏难度只需改这一个地方。

### 2. 盘子数量动态扩展机制
**位置**: `updatePlates()` 函数
- 1个盘子: 居中
- 2个盘子: 1/3 和 2/3 处
- 3个盘子: 1/4, 1/2, 3/4 处

**关键点**: 所有盘子共享同一个 `handleInput()` 移动逻辑，同步左右移动。

### 3. 礼盒生成与速度递增
**位置**: `spawnGift()` 函数
- 基础速度 × `speedMultiplier` (1 + score/2000)
- 概率: 普通60% / 高级30% / 稀有10%

### 4. 自定义图片存储方案
**位置**: `loadCustomImages()` / `saveCustomImages()`
- 使用 `FileReader.readAsDataURL()` 转 Base64
- 存储在 `localStorage` 的 `catchGift_images` 键中
- **注意**: Base64 图片较大，localStorage 通常限制 5-10MB

## 🎨 渲染系统

### 默认绘制方案（无自定义图片时）
| 元素 | 绘制方式 |
|------|----------|
| 盘子 | Canvas 椭圆 + 高光，卡通风格 |
| 礼盒 | 彩色方块 + 白色丝带 + 蝴蝶结 |
| 粒子 | 圆形 + 透明度渐变 + 重力模拟 |
| 背景 | 动态星星（正弦波控制大小变化） |

### 自定义图片加载逻辑
```javascript
if (customImages.plate) {
    const img = new Image();
    img.src = customImages.plate;
    if (img.complete) { drawImage(...) }
}
// fallback 到默认绘制
```
**注意**: 图片加载是异步的，这里用 `img.complete` 做简单检查。如果图片未加载完成，会暂时显示默认绘制。

## 🎯 碰撞检测

**位置**: `checkCollision()` 函数
- 使用 AABB (轴对齐边界框) 检测
- 礼盒是正方形，盘子是椭圆，实际检测用矩形近似
- **优化点**: 盘子高度较小，碰撞区域偏上更合理

## 2026-05-14 更新：盘子移动速度优化

### 改动内容

2. **长按加速机制**:
   - 新增 `plateSpeedBoost: 2`（2倍速）
   - 新增 `plateBoostDelay: 300ms`（长按触发阈值）
   - 新增 `gameState.keyPressTime` 记录按键按下时间戳
3. **加速逻辑**: 在 `handleInput()` 中检测任一方向长按超过 300ms，速度翻倍

### 技术细节
- 键盘和移动端触摸都支持长按检测
- 加速基于 `Date.now()` 时间差计算，与帧率无关
- 左右切换方向时不会重置加速状态（只要有一个方向在长按就保持加速）

## 2026-05-14 更新：多盘子排列优化

### 改动内容
1. **紧密并排排列**: 多个盘子改为紧密相邻，无间隙
2. **统一控制**: 整组盘子作为一个整体移动
3. **边界检测优化**: 以整组盘子的左右边界为基准，确保不会部分移出屏幕

### 技术细节
- `updatePlates()`: 计算总宽度 `totalWidth = plateCount * plateWidth`，然后居中排列
- `handleInput()`: 先计算整组的左右边界，再统一移动所有盘子
- 边界限制改为整组限制：左边缘 ≥ 0，右边缘 ≤ canvasWidth
